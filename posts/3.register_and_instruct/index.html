寄存器#

通用寄存器#

  • EAX
    累加寄存器（accumulator register），一般用来存放返回值
  • EBX
    基址寄存器（base register）
  • ECX
    计数寄存器（counter register），REP指令会直接使用这个ECX中的值
  • EDX
    数据寄存器（data register）

指针寄存器#

  • ESP
    堆栈指针寄存器（stack pointer register），指向当前（最新）堆栈数据的地址
  • EBP
    基址指针寄存器（base pointer register），一般指向栈底地址
  • EIP
    指令指针寄存器（instruction register），CPU将执行的下一条指令的地址
  • ESI
    源索引寄存器（source index register）
  • EDI
    目的索引寄存器（destination index register）

标志寄存器 FR#

  • 0_CF
    Carry Flag
    无符号溢出位，结果作为无符号数时，最高位有进位或者借位

  • 2_PF
    Parity Flag
    奇偶校验位，最低有效字节中有偶数个1，一般检验数据传输未发生错误

  • 4_AF
    Adjust Flag
    第三位有进位或借位，一般用于BCD（binary-code decimal）算术运算中

  • 6_ZF
    Zero Flag
    结果为0

  • 7_SF
    Sign Flag
    最高有效位的值是1还是0

  • 8_TF

  • 9_IF

  • 10_DF
    Direction Flag
    方向标志位，控制串指令（MOVS，CMPS，SCAS，LODS以及STOS）自减

  • 11_OF
    Overflow Flag
    有符号溢出位，结果作为有符号数时发生溢出

汇编指令#

MOV#

  • MOV A , B
    赋值指令，B赋值给A.
    C语言形式：

    int A,B;
A = B;

  • MOV ptr[A] , B
    A将作为地址，将B赋值到A.
    C语言形式：

    int *A,B;
*A = B;

MOV EAX , B与MOV ptr[EAX] , B的区别
MOV EAX , B：把B放到EAX这个寄存器.
MOV ptr[EAX] , B：把寄存器里的值作为地址，将B放到这个地址中.

ADD#

ADD A , B
A与B相加的结果赋值给A.
C语言形式：

int A,B;
A += B;
//or
A = A+B;

SUB#

SUB A , B
A减B的结果赋值给A.
C语言形式：

int A,B;
A -= B;
//or
A = A-B;

MOVS#

MOVSB、MOVSW、MOVSD
将ptr[ESI]赋值给ptr[EDI]，并且ESI、EDI的值自增或自减数据宽度
C语言形式：

int *pS, *pD;
*pS = *pD;
pS ++;
pD ++;

STOS#

STOSB、STOSW、STOSD
将EAX/AX/AL/AH赋值给ptr[EDI]，并且EDI的值自增或自减数据宽度
C语言形式：

int A, *pD;
*pD = A;
pD ++;

REP#

REP ?
重复执行?指令ECX次
C语言形式：

int ECX=0x10, a, b;
void mov(int* a, b)
{
  *a=b;
}

int main()
{
  for(i=0; i<ECX; i++)  //REP: ECX
  {
    mov(&a, b);  // ?
  }
}

PUSH#

PUSH im/r/m
ESP自减数据宽度，然后将数据赋值给ptr [ESP]中（压栈）
相当于：

SUB ESP , 1/2/4
MOV ptr[ESP] , ?

C语言形式：

int *ESP, A;
ESP -= sizeof A;
*ESP = A;

POP#

POP r/m
与PUSH相反，将数据从ptr [ESP]赋值给r/m，然后ESP自减数据宽度
相当于：

MOV ? , ptr[ESP]
ADD ESP , 1/2/4

C语言形式：

int *ESP, A;
A = *ESP;
ESP += sizeof A;

JMP#

JMP r/m/im
修改EIP的值，可以理解为MOV EIP , r/m/im，但是EIP是无法用MOV的

CALL#

CALL r/m/im
当前EIP入栈，修改EIP为r/m/im
可以理解为：

PUSH EIP
MOV EIP , r/m/im

RETN#

RETN (im)
与CALL相反，出栈到EIP，栈指针ESP自增im
可以理解为：

POP EIP
ADD ESP , im

CMP#

CMP A , B
相当于SUB但是不保存结果，一般用于比较两个数大小

TEST#

TEST A , A
相当于AND但是不保存结果，一般用于判断一个数是否为0

STD#

设置FR之DF=1，即串指令自减

CLD#

设置FR之DF=0，即串指令自增

JCC#

条件跳转（置EIP）

  • JZ（JE）
    ZF==1（结果是0）
  • JNZ（JNE）
    ZF==0（结果非0）
  • JB（JNAE）
    CF==1（作为无符号溢出，CMP/SUB时即借位小于）
  • JNA（JBE）
    CF==1 || ZF==1（非0且无符号未溢出，CMP/SUB时即小于或等于）
  • JA（JNBE）
    CF==0 && ZF==0（非0且无符号未溢出，CMP/SUB时即大于）
  • JNB（JAE）
    CF==0（作为无符号未溢出，CMP/SUB时即大于或等于）
  • JL（JNGE）
    SF!=OF（有符号数小于）
  • JLE（JNG）
    SF!=OF || ZF==1（有符号数小于或等于）
  • JG（JNLE）
    SF==OF && ZF==0（有符号数大于）
  • JGE（JNL）
    SF==OF（有符号数大于或等于）
  • JS
    SF==1（结果最高位是1）
  • JNS
    SF==0（结果最高位是0）
  • JO
    OF==1（作为有符号数溢出）
  • JNO
    OF==0（作为有符号数未溢出）
  • JP（JPE）
    PF==1
  • JNP（JPO）
    PE==0

堆栈传参#

传参与读参#

  1. PUSH参数n个
  2. CALL函数
  3. RETN
  4. ADD ESP , width*n

OR：

  1. PUSH参数n个
  2. CALL函数
  3. RETN width*n

堆栈情况：

函数返回地址参数_0参数_1…参数_i原堆栈
CALLPUSHPUSHPUSHPUSH…

参数寻址#

  • ESP寻址
    堆栈情况（32位）：

    函数返回地址参数_0参数_1…参数_i原堆栈
    CALLPUSHPUSHPUSHPUSH…
    ESPESP+4ESP+8…ESP+4+4iESP+4+4n
    函数执行时（CALL），ESP指向函数返回地址，ESP+4开始依次指向函数参数.
    函数结束时（RETN），ESP指向参数_0，所以需要恢复栈平衡，即ESP自增n个4（n个参数，一个参数是4字节）.
    栈平衡的恢复有两种：
    - 内平衡
    RETN 4N
    - 外平衡
    CALL后面ADD ESP , 4n

  • EBP寻址

    1. EBP压栈（备份原EBP）. *1
    2. EBP=ESP. *2
    3. ESP提升. *3

    堆栈情况（32位）：

    新栈数据…原EBP函数返回地址参数_0参数_1…参数_i原堆栈…原堆栈
    SUB ESP,X *3👆PUSH EBP *1👇MOV EBP,ESP *2CALLPUSHPUSHPUSHPUSH
    ESPEBP 原ESPEBP+4EBP+8ESP+C…EBP+8+4iEBP+8+4n原EBP
    函数执行时（CALL），完成上述三步.
    函数结束时（RETN）：
    1. 恢复栈顶ADD ESP,X
    2. 恢复EBPPOP EBP
    3. RETN+栈平衡
